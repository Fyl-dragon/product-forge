# PM Retro

Use after delivery, review, or major decision changes.

## Workflow

1. Read `../product-forge/references/interaction-protocol.md` and follow it strictly.
2. Before retrospective work, ask exactly one boundary question with exactly three options:
   - 问题：这次 retro 先沉淀哪类经验？
   - A. 决策和范围偏差。
   - B. 上线结果、指标和未解决问题。
   - C. 流程、模板和项目管理门禁改进。
   - 请回复 A / B / C。
3. After the user chooses, read `.product/STATE.md`, relevant feature pack, and recent artifacts.
4. Capture:
   - 做成了什么
   - 关键决策
   - 偏离原计划的地方
   - 学到的产品模式
   - 未解决问题
   - 后续需求
   - 指标复盘和真实结果
   - 对模板/流程的改进
   - 项目管理节奏、沟通和门禁改进
5. Update or propose updates to `.product/STATE.md` and feature `versions/` only when the user wants the retro persisted.
6. Preserve lessons that future `pm-*` work must reuse.
