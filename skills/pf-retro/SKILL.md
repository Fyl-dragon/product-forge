# PF Retro

Use after delivery, review, or major decision changes.

## Workflow

1. Read `.product/STATE.md`, relevant feature pack, and recent artifacts.
2. Capture:
   - 做成了什么
   - 关键决策
   - 偏离原计划的地方
   - 学到的产品模式
   - 未解决问题
   - 后续需求
   - 指标复盘和真实结果
   - 对模板/流程的改进
   - 项目管理节奏、沟通和门禁改进
3. Update or propose updates to `.product/STATE.md` and feature `versions/`.
4. Preserve lessons that future `pf-*` work must reuse.
