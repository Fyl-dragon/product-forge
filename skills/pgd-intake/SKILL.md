# PGD Intake

Use for early requirement analysis.

## Workflow

1. Read `.product/PRODUCT.md`, `.product/REQUIREMENTS.md`, and relevant feature context when available.
2. Separate confirmed facts, inferred assumptions, proposed solution, and unresolved questions.
3. Produce a structured intake:
   - 背景
   - 目标用户
   - 业务目标
   - 问题定义
   - 核心场景
   - 范围内
   - 范围外
   - 关键规则
   - 风险
   - 待确认问题
4. If the request is actionable, append or propose a requirement entry for `.product/REQUIREMENTS.md`.

## Quality Bar

Do not jump directly to UI. Lock problem, user, success criteria, and boundary first.
